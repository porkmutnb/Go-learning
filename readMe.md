## 📖 ข้อมูลควรรู้เกี่ยวกับภาษา Go (Golang)

ภาษา Go ถูกออกแบบโดยทีมวิศวกรจาก **Google** เพื่อแก้ปัญหาความยุ่งยากในระบบสเกลใหญ่ มีลักษณะเด่นดังนี้ครับ:

### ⚡ Performance & Simplicity

- **Compiled Language:** Compile เป็นไฟล์ Binary ตัวเดียวจบ ไม่ต้องพึ่งพา JVM เหมือน Java
- **Strict Compiler:** ถ้าประกาศตัวแปรไว้แล้วไม่ได้ใช้ จะไม่อนุญาตให้รันเด็ดขาด เพื่อความสะอาดของโค้ด

### 🏗️ Structure & OOP (The Go Way)

- **Composition over Inheritance:** Go ไม่มีคลาสและไม่มีการสืบทอดแบบ Java แต่จะใช้ **Structs** และ **Interfaces** แทน
- **Methods:** ผูกฟังก์ชันเข้ากับ Struct ได้ผ่าน **Receiver Function**

### 🛰️ Concurrency (The Rocket Power)

- **Goroutines:** ใช้ทรัพยากรน้อยมาก (Lightweight) รันงานเป็นแสนชิ้นพร้อมกันได้สบาย
- **Channels:** ใช้ท่อส่งข้อมูลในการสื่อสารระหว่าง Goroutines เพื่อความปลอดภัย (Concurrency Safety)

### 🛡️ Robustness

- **Explicit Error Handling:** ไม่ใช้ `try-catch` แต่เช็ค `if err != nil` ทันที ทำให้ระบบเสถียร

## 🛠️ Cheat Sheet: คำสั่งที่ควรรู้

### 🐹 Go CLI Basic

| คำสั่ง               | ความหมาย / การใช้งาน                                               |
| :------------------- | :----------------------------------------------------------------- |
| `go mod init <name>` | เริ่มต้นโปรเจกต์ใหม่ (เหมือนสร้าง `pom.xml` ใน Maven)              |
| `go run <file>.go`   | สั่ง Compile และรันโปรแกรมทันที (ใช้ตอน Develop)                   |
| `go build`           | Compile เป็นไฟล์ Binary (.exe) เพื่อนำไปใช้งานจริง                 |
| `go mod tidy`        | จัดระเบียบโมดูล ลบตัวที่ไม่ได้ใช้ และเพิ่มตัวที่ขาดให้โดยอัตโนมัติ |
| `go fmt`             | จัด Format โค้ดให้สวยงามตามมาตรฐาน Go                              |

### 🌿 Git Basic

| คำสั่ง                | ความหมาย / การใช้งาน                           |
| :-------------------- | :--------------------------------------------- |
| `git status`          | ตรวจสอบสถานะไฟล์ว่ามีอะไรแก้ไขบ้าง             |
| `git add .`           | เตรียมไฟล์ทั้งหมดเพื่อรอการบันทึก (Stage)      |
| `git commit -m "msg"` | บันทึกการเปลี่ยนแปลงพร้อมข้อความอธิบาย         |
| `git push`            | ดันโค้ดจากเครื่องเราขึ้นไปบน GitHub            |
| `git pull`            | ดึงโค้ดล่าสุดจาก GitHub ลงมาอัปเดตงานในเครื่อง |

## 📂 รายละเอียด 15 บทเรียนพื้นฐาน (Basic Level)

| บทเรียน | หัวข้อการเรียนรู้ | รายละเอียดโดยย่อ                                           |
| :------ | :---------------- | :--------------------------------------------------------- |
| **01**  | `Hello Go`        | การติดตั้งและ Compile/Run ครั้งแรก                         |
| **02**  | `Interaction`     | การจัดการเวลา (Time) และการ Format เวลาที่เป็นเอกลักษณ์    |
| **03**  | `Decision`        | การใช้ If-Else และการรับค่าจาก Keyboard ด้วย Scanln        |
| **04**  | `For Loop`        | การวนซ้ำหนึ่งเดียวใน Go ที่ครอบจักรวาล                     |
| **05**  | `Functions`       | การเขียนฟังก์ชันที่คืนค่าได้หลายตัวพร้อมกัน (Multi-Return) |
| **06**  | `Slices`          | การใช้งาน Dynamic Array ที่ยืดหยุ่นและทรงพลัง              |
| **07**  | `Maps`            | การเก็บข้อมูลแบบ Key-Value สไตล์พจนานุกรม                  |
| **08**  | `Structs`         | การสร้างโครงสร้างข้อมูลเพื่อใช้แทนคลาส                     |
| **09**  | `Methods`         | การเพิ่มความสามารถ (Behavior) ให้กับ Struct                |
| **10**  | `Pointers`        | การจัดการที่อยู่ตัวแปร (Address) และการส่งค่าแบบอ้างอิง    |
| **11**  | `Interfaces`      | การใช้ "สัญญาใจ" เพื่อสร้างระบบที่ยืดหยุ่น (Decoupling)    |
| **12**  | `Goroutines`      | เริ่มต้นการทำงานแบบอซิงโครนัส (Asynchronous)               |
| **13**  | `WaitGroups`      | การสั่งให้ Main รอน้อง ๆ (Goroutines) ทำงานให้เสร็จ        |
| **14**  | `Channels`        | การส่งของข้ามท่อระหว่างงานเบื้องหลัง                       |
| **15**  | `Error Handling`  | การดักจับปัญหาอย่างเป็นระบบและปลอดภัย                      |
